# 2. Stack Tecnológico

### 2.1 Stack tecnológico

| Camada | Tecnologia | Papel |
| ---| ---| --- |
| Frontend | Next.js (App Router) + React | Interface do usuário |
| API | Python (FastAPI ou Flask — a definir pelo time) | Endpoints REST do backend |
| Banco de dados | PostgreSQL | Persistência de dados |
| IA — Parsing | Claude API (Anthropic) | Extração de movimentações de PDFs e arquivos |
| ERP externo | Omie API | Fonte de dados dos clientes (lançamentos, contas, fornecedores) |
| Cache nível 1 | PostgreSQL (`omie_accounts_cache`) | Cache de contas correntes por cliente (TTL 24h) |
| Cache nível 2 | Map em memória ou Redis | Cache de lançamentos Omie durante sessão de revisão (TTL 2h) |

### 2.3 Fluxo geral de dados

```java
Usuário faz upload do arquivo
    ↓
Frontend calcula SHA-256 e valida formato/tamanho (no browser)
    ↓
Backend consulta duplicata no banco (hash + conta + mês)
    ↓
Backend envia arquivo para Claude API → recebe lista de transações
    ↓
Frontend exibe prévia das 5 primeiras linhas para confirmação
    ↓
Usuário confirma → Backend cria sessão + persiste entradas (com criptografia)
    ↓
Background: busca lançamentos Omie (3 endpoints) → executa cruzamento
    ↓
Background: classifica cada linha (conciliado / sem_omie / ignorado)
    ↓
Background: cria anomalias estruturais automáticas
    ↓
Frontend faz polling → detecta status 'reviewing' → redireciona para revisão
    ↓
Usuário revisa 4 abas (Movimentações / Divergências / Anomalias / Resumo)
    ↓
Usuário exporta relatório Excel (5 abas)
```

### 2.4 Responsabilidades: cliente vs servidor
Uma fronteira crítica de segurança:
**O que roda no browser (cliente):**
*   Cálculo de SHA-256 do arquivo (Web Crypto API) — apenas o hash é enviado ao servidor para verificação de duplicata
*   Validação de formato e tamanho do arquivo
*   Exibição e interação com todas as telas
*   Polling de status de processamento
**O que roda no servidor Python (API):**
*   Criptografia e descriptografia com AES-256-GCM
*   Todas as chamadas à API do Omie (com credenciais descriptografadas em memória)
*   Todas as chamadas à Claude API
*   Validações de negócio e autorização
*   Geração do Excel
**Nunca no cliente:**
*   Credenciais Omie (mesmo descriptografadas)
*   Chave de criptografia
*   API Key da Anthropic
