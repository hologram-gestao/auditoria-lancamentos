# 3. Modelo de Dados

### 3.1 Visão geral das tabelas
O banco armazena apenas dados operacionais da Hologram e referências ao Omie. **Nenhum dado identificável de clientes finais é armazenado sem criptografia.** CNPJ, contas bancárias, lançamentos, fornecedores e categorias são sempre buscados do Omie em tempo real.

```java
users                          ← usuários internos da Hologram
clients                        ← clientes BPO (nome + credenciais Omie criptografadas)
client_assignments             ← vínculo cliente ↔ gerente responsável
omie_accounts_cache            ← cache de contas correntes por cliente (TTL 24h)
reconciliation_sessions        ← sessões de conciliação (referências ao Omie)
reconciliation_file_entries    ← linhas do arquivo (desc. e notas criptografadas, valores em claro)
reconciliation_omie_entries    ← lançamentos Omie sem correspondente (apenas ID Omie)
anomaly_types                  ← catálogo de tipos de anomalia (seed + gerenciável pelo Admin)
reconciliation_anomalies       ← anomalias detectadas por sessão
```

### 3.2 Campos criptografados vs campos em claro

| Tabela | Campo | Criptografado? | Motivo |
| ---| ---| ---| --- |
| `clients` | `omie_app_key_encrypted` | Sim — AES-256-GCM | Credencial de acesso ao ERP do cliente |
| `clients` | `omie_app_secret_encrypted` | Sim — AES-256-GCM | Credencial de acesso ao ERP do cliente |
| `reconciliation_file_entries` | `description_encrypted` | Sim — AES-256-GCM | Pode conter nomes, CPFs, razões sociais |
| `reconciliation_file_entries` | `user_note_encrypted` | Sim — AES-256-GCM | Observação de conteúdo livre do analista |
| `reconciliation_file_entries` | `amount` | Não | Número decimal isolado — não identificável |
| `reconciliation_file_entries` | `balance` | Não | Número decimal isolado — não identificável |
| `reconciliation_file_entries` | `transaction_date` | Não | Necessário para ordenação e filtros SQL |
| `reconciliation_omie_entries` | `user_note_encrypted` | Sim — AES-256-GCM | Observação de conteúdo livre |
| `reconciliation_omie_entries` | `omie_lancamento_id` | Não | ID numérico — sem valor sem as credenciais |
| `reconciliation_anomalies` | `context_encrypted` | Sim — AES-256-GCM | Contexto pode conter descrições financeiras |
| `reconciliation_anomalies` | `resolution_note_encrypted` | Sim — AES-256-GCM | Observação de conteúdo livre |

###  3.3 O que NÃO é armazenado no banco local
*   Nome do cliente final (CNPJ, razão social) — buscado do Omie via `omie_empresa_id` + credenciais
*   Nome da conta bancária — buscado do Omie via `omie_conta_id`
*   Nome de fornecedores — buscado do Omie via `omie_lancamento_id`
*   Categorias dos lançamentos — buscado do Omie via `omie_lancamento_id`
*   Valores dos lançamentos Omie — buscado do Omie via `omie_lancamento_id`
*   O arquivo de extrato/fatura original — processado em memória e descartado
