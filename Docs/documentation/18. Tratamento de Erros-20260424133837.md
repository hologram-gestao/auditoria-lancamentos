# 18. Tratamento de Erros

### 18.1 Erros na camada de API (backend)
**Padrão de resposta de erro:**

```json
{
  "error": {
    "code": "CODIGO_DO_ERRO",
    "message": "Mensagem legível para o desenvolvedor",
    "userMessage": "Mensagem legível para o usuário final"
  }
}
```

**Códigos de erro relevantes:**

| Código | HTTP | Situação |
| ---| ---| --- |
| `UNAUTHORIZED` | 401 | Token ausente ou inválido |
| `TOKEN_EXPIRED` | 401 | Access token expirado (frontend deve tentar refresh) |
| `FORBIDDEN` | 403 | Usuário sem permissão para o recurso |
| `NOT_FOUND` | 404 | Recurso não encontrado |
| `DUPLICATE_FILE` | 409 | Arquivo já processado para esta conta e mês |
| `OMIE_AUTH_ERROR` | 502 | Credenciais Omie inválidas |
| `OMIE_TIMEOUT` | 504 | Omie não respondeu no tempo esperado |
| `PARSE_ERROR` | 422 | Claude API não conseguiu extrair movimentações |
| `VALIDATION_ERROR` | 400 | Dados inválidos na requisição |

###  18.2 Erros exibidos ao usuário (frontend)
**Toast notifications** (canto superior direito, desaparecem após 5s):
*   Sucesso: fundo verde — "Cliente criado com sucesso.", "Sessão de conciliação iniciada."
*   Erro genérico: fundo vermelho — "Ocorreu um erro inesperado. Tente novamente."
*   Aviso: fundo amarelo — "Alterações salvas. O gerente anterior perdeu acesso."
**Mensagens inline** (abaixo do campo ou botão que causou o erro):
*   Validação de formulário: mensagem específica por campo
*   Erro de conexão Omie: abaixo do botão "Testar conexão"
**Telas de erro dedicadas:**
*   Sessão com `status = 'error'`: tela de progresso exibe a `error_message` da sessão e botão "Voltar ao formulário"
*   Sessão não encontrada (404): página com mensagem "Conciliação não encontrada." e link "Voltar para {nome do cliente}"
### 18.3 Comportamento em perda de conexão
*   O frontend deve detectar falhas de rede e exibir banner: "Sem conexão com a internet. Verifique sua rede."
*   O polling de status da sessão deve suspender automaticamente e retomar quando a conexão for restaurada
*   Operações de escrita (PATCH, POST) devem exibir estado de erro e permitir retry manual
