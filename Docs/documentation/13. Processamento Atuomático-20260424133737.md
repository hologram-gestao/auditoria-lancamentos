# 13. Processamento Atuomático

### 13.1 Tela de progresso — `/clientes/{client_id}/conciliacao/{session_id}/processando`
**Quando aparece:** imediatamente após o usuário clicar em "Confirmar e processar".
**Elementos visíveis:**
*   Título: "Processando conciliação..."
*   Subtítulo: nome do cliente · conta bancária · mês de referência
*   Steps visuais com status atual:
    1. ✅ Salvando dados (completo imediatamente — backend confirma na criação da sessão)
    2. ⏳ Buscando lançamentos no Omie (em andamento durante a chamada ao Omie)
    3. ○ Cruzando movimentações (pendente)
    4. ○ Finalizando (pendente)
*   Texto: "Aguarde. Isso pode levar até 2 minutos."
*   **Sem botão de cancelar** — o processamento é assíncrono no servidor e não pode ser interrompido via frontend
**Como o frontend acompanha o progresso:**
*   Faz polling a cada 3 segundos em `GET /api/reconciliations/{session_id}/status`
*   Atualiza os steps visuais conforme o status retornado
*   Timeout do frontend: 5 minutos. Se exceder: exibir "O processamento está demorando. Tente atualizar a página para verificar o status."
**Quando o processamento termina:**
*   `status = 'reviewing'`: redireciona automaticamente para `/clientes/{client_id}/conciliacao/{session_id}`
*   `status = 'error'`: exibe mensagem de erro com a `error_message` da sessão + botão "Voltar ao formulário"
### 13.2 O que o backend faz no processamento (background)
**Etapa 1 — Criar sessão e entradas:**
1. Cria registro em `reconciliation_sessions` com `status: 'processing'`
    *   Para cada transação confirmada pelo usuário: cria registro em `reconciliation_file_entriestransaction_date`: data em claro
    *   `amount`: valor decimal com sinal, em claro
    *   `balance`: saldo decimal, em claro
    *   `description_encrypted`: descrição criptografada com AES-256-GCM
    *   `situation`: `'sem_omie'` (padrão inicial)
2. Retorna `session_id` ao frontend imediatamente
3. Dispara as etapas seguintes em background (assincronamente)
**Etapa 2 — Buscar lançamentos realizados (ListarExtrato):**
1. Descriptografa credenciais do cliente em memória
2. Calcula período: `period_start - date_tolerance_days` até `period_end + date_tolerance_days`
3. Chama `ListarExtrato` com o período expandido e o `omie_conta_id`
4. Normaliza os dados: converte datas, aplica sinal pelo campo `cNatureza`
5. Mantém em memória para o cruzamento
6. Em falha: atualiza `status = 'error'` na sessão com mensagem descritiva
**Etapa 3 — Buscar previstos e atrasados (ListarContasPagar + ListarContasReceber):**
1. Chama `ListarContasPagar` com status `"ATRASADO"` (paginar até esgotar)
2. Chama `ListarContasPagar` com status `"PREVISTO"` (paginar até esgotar)
3. Repete para `ListarContasReceber`
4. Mantém lista consolidada em memória para uso após o cruzamento
**Etapa 4 — Algoritmo de cruzamento:**
O cruzamento percorre cada linha do arquivo e tenta encontrar um lançamento Omie correspondente.
Para cada linha do arquivo:
1. Filtrar lançamentos Omie que satisfaçam simultaneamente:
    *   `|amount_arquivo - amount_omie| ≤ 0.01` (diferença máxima de R$0,01)
    *   `|date_arquivo - date_omie| ≤ date_tolerance_days` (diferença máxima em dias)
    *   Lançamento Omie ainda não vinculado a outra linha (controlar via conjunto de IDs usados)
2. Se nenhum candidato: linha permanece com `situation = 'sem_omie'`
3. Se um ou mais candidatos: escolher o de data mais próxima (desempate). Marcar como usado.
4. Atualizar `reconciliation_file_entries`: `situation = 'conciliado'`, `omie_lancamento_id = ID do match`
Após percorrer todas as linhas:
*   Lançamentos Omie da Etapa 2 (realizados) não consumidos: **ignorar** — já estavam conciliados no Omie antes do período ou são de natureza diferente
*   Lançamentos Omie da Etapa 3 (previstos/atrasados) não consumidos: inserir em `reconciliation_omie_entries` — esses são os casos de "Omie sem arquivo"
**Etapa 5 — Criar anomalias estruturais:**
1. Para cada `reconciliation_file_entries` com `situation = 'sem_omie'`:
    *   Criar `reconciliation_anomalies` com `anomaly_type.code = 'missing_in_omie'` e `detected_by = 'ai'`
2. Para cada `reconciliation_omie_entries` com `omie_status = 'Atrasado'`:
    *   Criar `reconciliation_anomalies` com `anomaly_type.code = 'missing_in_file'` e `detected_by = 'ai'`
3. Atualizar `reconciliation_sessions.anomaly_count` com o total de anomalias criadas
**Etapa 6 — Finalizar:**
1. Calcular e atualizar em `reconciliation_sessions`:
    *   `balance_start`: saldo inicial retornado pela Claude API
    *   `balance_end_file`: saldo final calculado
    *   `balance_end_omie`: saldo conforme Omie (via endpoint de resumo)
    *   `balance_difference`: diferença entre os dois
    *   `conciliated_count`: contagem de linhas com `situation = 'conciliado'`
    *   `sem_omie_count`: contagem de linhas com `situation = 'sem_omie'`
    *   `omie_sem_arquivo_count`: contagem de registros em `reconciliation_omie_entries`
    *   `total_file_entries`: total de linhas do arquivo
    *   `status`: `'reviewing'`
    *   `processed_at`: timestamp atual
