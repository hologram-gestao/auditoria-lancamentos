# 9. Gestão de Clientes BPO

### 9.1 Tela de lista de clientes — `/clientes`
Esta é a tela inicial após o login. É a primeira tela que o usuário vê ao acessar o sistema.
**Elementos visíveis:**
*   Header com logo do Sistema de Auditoria de Lançamentos à esquerda e nome do usuário logado + botão de logout à direita
*   Sidebar/menu lateral com: "Clientes" (ativo), "Configurações" (apenas Admin)
*   Título da página: "Clientes"
*   Botão "Novo Cliente" no canto superior direito (visível para Admin e Gerente)
*   Campo de busca: filtra por nome do cliente (debounce 300ms)
**Tabela de clientes:**
*   Colunas para **Admin**: Nome · Gerente Responsável · Status · Conciliações · Cadastrado em · Ações
*   Colunas para **Gerente**: Nome · Status · Conciliações · Cadastrado em · Ações
*   Coluna "Conciliações": número de sessões de conciliação já realizadas para o cliente
*   Coluna "Status": badge verde "Ativo" ou badge vermelho "Inativo"
*   Coluna "Ações": ícone de olho (ver detalhe) e ícone de lápis (editar)
**Estado vazio:**
*   Sem clientes cadastrados: ícone + "Nenhum cliente cadastrado. Crie o primeiro cliente clicando em 'Novo Cliente'."
*   Busca sem resultado: "Nenhum cliente encontrado para '{termo}'."
**Paginação:** seletor de 10/20/50 por página + indicador "Página X de Y"
**Clique na linha ou no ícone de olho:** navega para `/clientes/{client_id}`
### 9.2 Modal: Cadastro de cliente
**Como abrir:** botão "Novo Cliente" na lista de clientes.
**Elementos do modal:**
*   Título: "Novo Cliente"
*   Campo "Nome do cliente" (text, obrigatório) — nome como a Hologram se refere ao cliente internamente
*   Campo "App Key Omie" (password, obrigatório) — toggle de visibilidade com ícone de olho
*   Campo "App Secret Omie" (password, obrigatório) — toggle de visibilidade com ícone de olho
*   Text-button "Testar conexão" (link/botão terciário, sem destaque) — posicionado abaixo dos campos de credenciais
**Estados do "Testar conexão":**
1. **Inicial:** texto azul clicável "Testar conexão"
2. **Durante o teste:** spinner + "Testando..." (campos e outros botões desabilitados durante o teste)
3. **Sucesso:** ícone de check verde + "✓ Conexão verificada com sucesso" (aparece inline, persiste)
4. **Falha:** ícone de X vermelho + mensagem de erro do Omie (ex: "Credenciais inválidas") + "Testar conexão" volta ao estado inicial para nova tentativa
**Botão "Salvar":**
*   Desabilitado enquanto "Testar conexão" não tiver sido executado com sucesso
*   Desabilitado se qualquer campo obrigatório estiver vazio
*   Após clicar: loading state, chama API de criação, fecha modal, toast de sucesso
**O que acontece no backend ao salvar:**
1. Recebe nome, app\_key e app\_secret em texto plano
2. Gera IV aleatório
3. Criptografa app\_key e app\_secret com AES-256-GCM
4. Salva em `clients` com os campos criptografados, IV e tag
5. Cria registro em `client_assignments` com `user_id = usuário autenticado`
6. As credenciais originais são descartadas — nunca persistidas em aberto
### 9.3 Modal: Edição de cliente
**Como abrir:** ícone de lápis na lista ou botão "Editar" na tela de detalhe.
**Elementos:**
*   Campo "Nome do cliente" pré-preenchido e editável
*   Campo "App Key Omie" vazio com placeholder `••••••••` — não mostra o valor atual
*   Campo "App Secret Omie" vazio com placeholder `••••••••` — não mostra o valor atual
*   Text-button "Testar conexão" (mesmo comportamento do cadastro)
*   Campo "Status" (select): Ativo/Inativo
*   **Somente Admin:** seção adicional "Gerente Responsável" com select de todos os gerentes ativos
**Lógica de atualização das credenciais:**
*   Se o usuário deixar os campos de credenciais vazios → apenas o nome e o status são atualizados. As credenciais existentes no banco são mantidas.
*   Se o usuário preencher os campos de credenciais → "Testar conexão" fica obrigatório antes de salvar. As credenciais são recriptografadas com novo IV.
**Reatribuição (somente Admin):**
*   Select de gerente muda o responsável
*   Ao salvar: `client_assignments.user_id` é atualizado para o novo gerente
*   O gerente anterior perde acesso imediatamente (próxima requisição com `user_id` antigo retorna 403)
