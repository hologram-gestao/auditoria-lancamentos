# 8. Gestão de Usuários

### 8.1 Acesso
Rota: `/configuracoes/usuarios`
Visível **apenas para Admin**. Manager não vê este item no menu de configurações. Tentativa de acesso direto à URL por Manager retorna HTTP 403 e redireciona para `/clientes`.
### 8.2 Tela de listagem
**Layout:** tabela com busca no topo.
**Elementos visíveis:**
*   Breadcrumb: "Configurações > Usuários"
*   Campo de busca: texto livre, filtra por nome ou e-mail em tempo real (debounce 300ms, chama API a cada digitação)
*   Botão "Novo Usuário" no canto superior direito
*   Tabela com colunas: Nome · E-mail · Perfil (Admin/Gerente) · Status (Ativo/Inativo) · Data de cadastro · Ações
*   Paginação na parte inferior: "Mostrando X–Y de Z" + seletor de página
*   Estado vazio (sem usuários ou busca sem resultado): ícone + mensagem "Nenhum usuário encontrado"
**Coluna Perfil:**
*   Admin: badge azul com texto "Admin"
*   Gerente: badge cinza com texto "Gerente"
**Coluna Status:**
*   Ativo: badge verde "Ativo"
*   Inativo: badge vermelho "Inativo" + linha levemente opacificada
**Coluna Ações:**
*   Ícone de lápis (editar): abre modal de edição
*   Ícone de toggle: ativa/desativa o usuário (com modal de confirmação)
*   O Admin logado não vê o ícone de toggle na sua própria linha
### 8.3 Modal: Novo Usuário
**Elementos:**
*   Título: "Novo Usuário"
*   Campo "Nome completo" (text, obrigatório)
*   Campo "E-mail" (email, obrigatório) — validar unicidade ao submeter
*   Campo "Perfil" (select, obrigatório): opções "Admin" e "Gerente"
*   Campo "Senha inicial" (password, obrigatório) com toggle de visibilidade — mínimo 8 caracteres
*   Botão "Cancelar" (secundário) e "Criar usuário" (primário)
*   Loading state no botão ao salvar
*   Erro de e-mail duplicado: mensagem inline abaixo do campo "Este e-mail já está em uso."
**Após criar com sucesso:**
*   Modal fecha
*   Lista atualiza automaticamente (sem reload de página)
*   Toast de confirmação: "Usuário criado com sucesso."
### 8.4 Modal: Editar Usuário
**Elementos:**
*   Título: "Editar Usuário"
*   Campos pré-preenchidos: Nome, E-mail, Perfil
*   Sem campo de senha — alteração de senha não está no escopo do MVP
*   Admin não consegue alterar o próprio perfil para "Gerente" (campo desabilitado na sua própria linha)
*   Botão "Cancelar" e "Salvar alterações"
### 8.5 Desativar usuário
**Fluxo:**
1. Admin clica no ícone de toggle de um usuário ativo
2. Modal de confirmação: "Deseja desativar \[nome\]? Ele perderá acesso imediatamente."
3. Botões: "Cancelar" e "Desativar" (destrutivo, vermelho)
4. Após confirmar: `active = false` no banco, usuário perde acesso imediatamente (middleware verifica `active` a cada request)
5. Toast: "Usuário desativado."
