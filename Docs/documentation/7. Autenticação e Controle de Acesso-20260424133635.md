# 7. Autenticação e Controle de Acesso

### 7.1 Tela de Login — `/login`
**Visibilidade:** única tela pública do sistema. Qualquer usuário autenticado que acesse `/login` é redirecionado para `/clientes`.
**O que o usuário vê:**
*   Logo/nome do Sistema de Auditoria de Lançamentos centralizado no topo
*   Campo de e-mail com label "E-mail" (type email, obrigatório)
*   Campo de senha com label "Senha" (type password, obrigatório) e ícone de olho para toggle de visibilidade
*   Botão "Entrar" (primário, largura total)
*   Nenhum link de "Esqueci minha senha" — recuperação de acesso é feita pelo Admin diretamente
**Comportamento do botão:**
*   Desabilitado enquanto e-mail ou senha estiverem vazios
*   Exibe spinner e texto "Entrando..." durante a chamada
*   Reabilitado imediatamente em caso de erro
**Estados de erro:**
*   Credenciais inválidas: mensagem inline abaixo do botão — "E-mail ou senha incorretos" (sem especificar qual). A resposta da API é propositalmente genérica.
*   Rate limit atingido: "Muitas tentativas. Aguarde 1 minuto antes de tentar novamente."
*   Erro de rede: "Não foi possível conectar ao servidor. Verifique sua conexão."
**O que acontece ao autenticar:**
1. Frontend recebe access token (JWT, 1h) e refresh token (7 dias) no response
2. Armazena ambos em cookies HttpOnly
3. Redireciona para `/clientes`
**Middleware de proteção:**
*   Todas as rotas dentro de `/(app)/*` verificam a presença de access token válido
*   Token expirado: verificar se refresh token é válido → se sim, renovar silenciosamente e continuar → se não, redirecionar para `/login`
*   Token ausente: redirecionar imediatamente para `/login`
### 7.2 Renovação de token (transparente ao usuário)
O frontend deve implementar um interceptor de requisições que:
1. Detecta resposta HTTP 401 com código `TOKEN_EXPIRED`
2. Chama `POST /api/auth/refresh` com o refresh token
3. Recebe novo access token
4. Repete a requisição original com o novo token
5. Se o refresh token também expirou (HTTP 401): redireciona para `/login`
O usuário não percebe este processo — apenas se ambos os tokens expirarem (após 7 dias de inatividade).
