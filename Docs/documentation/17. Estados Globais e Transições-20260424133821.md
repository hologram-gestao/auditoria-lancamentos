# 17. Estados Globais e Transições

### 17.1 Estados de uma sessão de conciliação

```bash
processing → reviewing → done
     ↓
   error
```

*   **processing:** sessão criada, processamento em andamento (background jobs em execução)
*   **reviewing:** processamento concluído, aguardando revisão do analista
*   **done:** analista concluiu a revisão (estado final — atingido quando o analista exporta o relatório ou marca como concluída)
*   **error:** falha irrecuperável durante o processamento
> **Nota:** não há transição automática de `reviewing` para `done`. O analista decide quando está satisfeito com a revisão. A exportação do relatório pode ocorrer em qualquer estado após `reviewing`.
### 17.2 Estados de uma linha do arquivo

```scss
sem_omie → conciliado
sem_omie → ignorado
conciliado → ignorado
ignorado → sem_omie (restaurar)
ignorado → conciliado (restaurar, se havia match)
```

*   `sem_omie`: estado inicial para todas as linhas. Pode mover para `conciliado` (match encontrado no cruzamento automático) ou `ignorado` (analista decide não reconciliar).
*   `conciliado`: linha com match Omie encontrado. Pode ser mudada para `ignorado` ou ter o match substituído pelo "Trocar".
*   `ignorado`: decisão do analista de não reconciliar. Linha não conta nos indicadores de pendência.
### 17.3 Estados de uma anomalia

```plain
pendente → resolvida
```

Não há retorno de resolvida para pendente — a resolução é permanente (com nota obrigatória).
